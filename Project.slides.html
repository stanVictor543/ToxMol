Predicia Toxicitii Moleculare folosind Machine Learning

Studiu de caz: Setul de date Tox21 i Receptorul Androgenic (NR-AR)

1. Introducere

n domeniul descoperirii medicamentelor i al toxicologiei computaionale, capacitatea de a prezice dac o molecul nou va fi toxic nainte de sinteza fizic este crucial. Acest proiect utilizeaz tehnici de Chemoinformatic i Machine Learning pentru a clasifica compuii chimici n funcie de activitatea lor toxicologic.

Setul de date: Vom folosi Tox21 (Toxicology in the 21st Century), un set de date de referin care conine mii de compui testai in vitro pentru diverse efecte biologice.

Obiectivul: Ne concentrm pe un obiectiv specific: NR-AR (Nuclear Receptor - Androgen Receptor). Vom construi modele capabile s prezic dac o molecul activeaz sau inhib acest receptor, ceea ce poate indica poteniale efecte adverse endocrine.

2. Reprezentarea Datelor: De la Structur la Vectori

Calculatoarele nu pot "nelege" direct o structur chimic desenat. Trebuie s o convertim ntr-un format numeric.

A. SMILES (Simplified Molecular Input Line Entry System) Este o notaie liniar folosind caractere ASCII pentru a reprezenta structura chimic.

  • Exemplu (Etanol): CCO
  • Exemplu (Benzen): c1ccccc1

B. Morgan Fingerprints (Amprente Moleculare) Pentru a aplica algoritmi de ML, transformm irurile SMILES n vectori de lungime fix. Folosim Morgan Fingerprints (similare cu ECFP - Extended-Connectivity Fingerprints).

Algoritmul analizeaz vecintatea fiecrui atom pe o anumit raz ($R$) i aplic funcii de hashing pentru a genera un vector binar.

Matematic, o molecul $M$ este reprezentat ca un vector $V$:

$$V_{M} = [b_1, b_2, \dots, b_N]$$

Unde:

  • $N$ este lungimea vectorului (n cazul nostru, $N=2048$ bii).
  • $b_i \in \{0, 1\}$. Valoarea $1$ indic prezena unei substructuri specifice, iar $0$ absena ei.

def clean_and_load_data(data_dir):
    """ncarc datele i cur automat cache-ul n caz de eroare."""
    try:
        print(f"[*] Se ncarc datele Tox21 n: {data_dir}...")
        # DeepChem ncarc automat train/valid/test split-uri
        return dc.molnet.load_tox21(data_dir=data_dir)
    except Exception as e:
        print(f"[!] Eroare detectat la ncrcare: {e}")
        if os.path.exists(data_dir):
            print("[-] Se cur cache-ul corupt...")
            shutil.rmtree(data_dir)
            print("[+] Cache curat. Rencercare...")
            return dc.molnet.load_tox21(data_dir=data_dir)
        raise e


def smiles_to_fp(smiles):
    """Conversie sigur SMILES -> Morgan Fingerprint (Vector Binar)."""
    try:
        mol = Chem.MolFromSmiles(smiles)
        if mol:
            return morgan_gen.GetFingerprintAsNumPy(mol)
    except:
        return None
    return None


def preprocess_dataset(dataset, target_index):
    """
    Cur setul de date:
    1. Extrage etichetele (y) pentru task-ul int.
    2. Elimin intrrile unde eticheta este NaN (nu au fost testate).
    3. Convertete SMILES n Fingerprints.
    """
    print(f"[*] Procesare set date (iniial: {len(dataset)} molecule)...")
    y_all = dataset.y[:, target_index]
    smiles_all = dataset.ids
    
    valid_x, valid_y = [], []
    processed_count = 0
    
    for smiles, y in zip(smiles_all, y_all):
        if not np.isnan(y):
            fp = smiles_to_fp(smiles)
            if fp is not None:
                valid_x.append(fp)
                valid_y.append(int(y))
        
        processed_count += 1
        if processed_count % 2000 == 0:
            print(f"   -> Procesate: {processed_count}/{len(dataset)}")
                
    print(f"[+] Finalizat. Molecule valide rmase: {len(valid_x)}")
    return np.array(valid_x), np.array(valid_y)

4. Metodologie i Modele

Vom transforma acum datele folosind funciile definite anterior i vom antrena dou modele de clasificare diferite.

Modelele Utilizate:

  1. Bernoulli Naive Bayes (BNB):

    • Un model probabilistic bazat pe Teorema lui Bayes, potrivit pentru date binare (cum sunt amprentele noastre).
    • Formula: Calculeaz probabilitatea ca o molecul (reprezentat de vectorul $X$) s aparin clasei toxice $C_1$: $$P(C_1 | X) \propto P(C_1) \prod_{i=1}^{N} P(x_i | C_1)$$
    • De ce? Este rapid, simplu i funcioneaz surprinztor de bine pe date de dimensiuni mari.

  2. K-Nearest Neighbors (KNN):

    • Un algoritm bazat pe instane care clasific o molecul nou pe baza celor mai apropiai $K$ vecini din setul de antrenare.
    • n chimie, "apropierea" se msoar adesea folosind Distana Jaccard (Tanimoto) ntre vectorii binari $A$ i $B$: $$J(A, B) = \frac{|A \cap B|}{|A \cup B|} = \frac{\text{Bii comuni de 1}}{\text{Total bii de 1 unici}}$$

Vom folosi GridSearchCV pentru a gsi hiperparametrii optimi (ex: numrul $K$ de vecini).

5. Evaluarea Performanei

Pentru a evalua modelele, folosim mai multe metrici, deoarece acurateea singur este neltoare n cazul datelor dezechilibrate.

Metrici Cheie:

  1. AUC-ROC (Area Under ROC Curve): Msoar capacitatea modelului de a distinge ntre clase la diferite praguri. 0.5 = ghicire aleatorie, 1.0 = perfect.
  2. F1-Score: Media armonic ntre Precizie i Recall. Este crucial cnd clasa pozitiv (toxic) este rar. $$F1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

Optimizarea Pragului (Threshold Tuning): Majoritatea clasificatorilor folosesc un prag implicit de 0.5 (dac probabilitatea > 50%, este toxic). Totui, pentru a maximiza detectarea toxicelor (Recall), putem ajusta acest prag. Vom optimiza pragul pentru KNN pentru a maximiza scorul F1.

def evaluate_metrics(model, X, y, threshold=0.5):
    """Calculare metrici i returnare probabiliti."""
    probs = model.predict_proba(X)[:, 1]
    preds = (probs >= threshold).astype(int)
    return {
        'AUC-ROC': roc_auc_score(y, probs),
        'F1': f1_score(y, preds),
        'Acc': accuracy_score(y, preds),
        'Precision': precision_score(y, preds, zero_division=0),
        'Recall': recall_score(y, preds),
        'Probs': probs,
        'Preds': preds
    }

from sklearn.metrics import precision_score, recall_score

# Evaluare Iniial (Prag 0.5)
res_nb = evaluate_metrics(best_nb, X_test, y_test)
res_knn = evaluate_metrics(best_knn, X_test, y_test)

# --- Optimizare Prag pentru KNN ---
# Calculm curba Precision-Recall
precision_k, recall_k, thresholds_k = precision_recall_curve(y_test, res_knn['Probs'])
# Calculm F1 pentru fiecare prag posibil
f1_scores_k = 2 * (precision_k * recall_k) / (precision_k + recall_k + 1e-9)
# Gsim pragul care maximizeaz F1
best_thr_idx = np.argmax(f1_scores_k)
best_thr_knn = thresholds_k[best_thr_idx]

# Re-evalum KNN cu noul prag
res_knn_opt = evaluate_metrics(best_knn, X_test, y_test, threshold=best_thr_knn)

print(f"Prag optimizat pentru KNN: {best_thr_knn:.4f} (F1 maxim: {f1_scores_k[best_thr_idx]:.4f})")
